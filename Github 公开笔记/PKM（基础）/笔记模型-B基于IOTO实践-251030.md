###### OTO 理论和实践之“📂文件夹”
- Output（利我）
	- 类别：
		- 项目
		- 工作流（模块笔记）：进行项目产生的"经验"，基于项目实际具体工作流
	- 📂 根目录：领域工作流.canvas、[[0 闪念笔记（💡灵感库）]]
	- 📂：领域笔记、绘图脑图、工作流扩展
- Tasks
	- 类别：
		- “周记.md”Tasks
			- 唯一目的，推进项目！(Output利我/Outcome利他)，喔别忘了，项目经验构造==领域工作流==(Output利我)
			- 其次：安排日程
	- 📂：领域组件、周记、年度规划和里程碑
- Outcome（利他）
	- 类别：
		- 已产出项目
		- 已分享工作流
		- 笔记方式用链接！例：[[项目#输出]]
		- 成果方式：视频、文章（都有对应的工作流方式） ^935c83
	- 📂：领域笔记（对应 Output 利我文件夹）

###### 问题解答
- 任务工作流：IOTO
- 任务来源：不知道下一个进度，下一个正确的任务？答：area.canvas
- 任务路径：[[☑️年度规划与里程碑🏆]]、周记（日程）、领域目标（重复任务）、项目专用的 Excalidraw
- 任务执行：手动输入计时，支持动态调整状态（编辑）
	- 放弃 Text Block Timer 计时，改成手动计时，这样不卡顿，好输入调整
